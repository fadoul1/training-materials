# Module 1 — Comprendre Copilot et pourquoi la configuration compte

**Duree : 1h 00m** | **Jour 1**

## Objectif

Etablir le modele mental avant de toucher a un fichier de configuration.

## Theorie (45 min)

Copilot est un nouveau membre talentueux de l'equipe — il ecrit du bon code mais ne connait rien de votre codebase, vos conventions ou votre architecture. Chaque suggestion qu'il fait est basee uniquement sur ce qui se trouve dans sa fenetre de contexte au moment de la requete. La personnalisation est la facon dont vous partagez ce contexte de maniere fiable et coherente.

### Sujets couverts

- Comment Copilot assemble le contexte et genere des suggestions
- Pourquoi les suggestions generiques se produisent et comment les eliminer
- Suggestions inline vs. mode chat — quand utiliser chaque mode
- Chat participants : `@workspace`, `@vscode`, `@terminal`
- Selection de modeles : modeles rapides pour les taches simples, modeles de raisonnement pour les decisions d'architecture
- Le chemin de configuration incremental : instructions → prompt files → skills → agents
- Demo avant/apres : un repository non-configure vs. un repository pleinement configure
