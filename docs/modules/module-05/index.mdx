# Module 5 — Custom Agents : personas specialisees avec scope controle

**Duree : 1h 00m** | **Jour 5**

## Objectif

Creer des personas IA specialisees avec un scope controle, un acces aux outils limite et la capacite d'orchestrer d'autres agents pour du travail multi-etapes.

## Theorie (30 min)

Un custom agent est une persona IA persistante definie dans un fichier Markdown. Il a son propre system prompt, son propre ensemble d'outils autorises, ses propres preferences de modele et ses propres limites de scope.

### Concepts cles

- **Restrictions d'outils** — vous pouvez lister explicitement quels outils l'agent peut utiliser
- **Orchestration de sous-agents** — un agent peut deleguer une sous-tache a un autre agent specialise

### Sujets couverts

- Structure Markdown d'un custom agent : frontmatter, system prompt, `tools`, `model`
- Comment ecrire un system prompt suffisamment specifique sans etre trop restrictif
- Restrictions explicites d'outils : pourquoi limiter les outils est une fonctionnalite
- Definition du scope : ce que l'agent doit faire et ce qu'il ne doit pas faire
- Orchestration de sous-agents : `delegate`, passage de contexte, reprise
- Archetypes d'agents : refactoring agent, migration agent, security audit agent
